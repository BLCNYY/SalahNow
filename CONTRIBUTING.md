# Contributing to Salah[Now]

Thanks for your interest in improving Salah[Now]. This document outlines how to set up the project, propose changes, and share feedback.

## Ways to contribute
- Report bugs and usability issues
- Request or design new features
- Improve accessibility, performance, or tests
- Add or refine translations
- Improve documentation

## Getting started
1. Fork the repository and clone your fork.
2. Install dependencies with `npm install`.
3. Run the development server with `npm run dev`.
4. Create a branch that describes your work, such as `feature/multi-language` or `fix/timezone-display`.
5. Keep changes focused and prefer small, reviewable pull requests.

## Development guidelines
- Use TypeScript and avoid `any` whenever possible.
- Keep components small and focused on a single concern.
- Reuse existing utilities and patterns before adding new ones.
- Ensure UI changes account for desktop and mobile experiences.
- Write accessible markup and verify keyboard navigation.

## Testing
- Run `npm run lint` before submitting changes.
- Manually verify your changes in development, including offline behavior when relevant.

## Pull requests
- Fill out the pull request template completely.
- Include clear screenshots or recordings for UI changes.
- Reference related issues and explain why the change is needed.
- Expect to revise your pull request based on reviewer feedback.

## Community expectations
- Follow the [Code of Conduct](CODE_OF_CONDUCT.md) in all interactions.
- Be constructive, respectful, and patient with maintainers and other contributors.
